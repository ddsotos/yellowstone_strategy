# Heuristic Bot Design

この文書は、強化学習の相手として使う deterministic heuristic bot の行動ルールを定義する。

目的は、学習対象プレイヤーが極端に不自然な相手だけを見て変な学習をしないように、完全に再現可能で、かつ最低限合理的な NPC を用意すること。

## 前提

- bot は `legal_actions(state)` で列挙された合法手だけから選ぶ。
- 同じ状態からは必ず同じ行動を返す。
- 乱数は使わない。
- ルールは上から順に適用する。
- 候補が複数残った場合は、次の固定 tie-break で1つに決める。

## 固定 Tie-break

配置アクション同士の tie-break は以下の順に小さいものを優先する。

1. `hand_index`
2. `position.x`
3. `position.y`
4. `frame.x`
5. `frame.y`

異なる action 種別が同点になった場合は、配置、1枚終了、補充の順に評価済みルールを優先する。補充アクション同士は `NEGATIVE_CARDS`, `DECK`, `NONE` の順にする。

## 補充フェーズのルール

1. ゲーム終了中なら行動しない。

2. 手札が0枚かつマイナスカードが6枚以上なら、マイナスカードから補充する。

   理由: マイナスカードから6枚補充すると、その6枚がマイナスカード置き場から取り除かれるため、次の決算で6点分の失点を避けられる。

3. 手札が6枚未満なら、山札から補充する。

4. 手札が6枚なら、補充しない。

## プレイフェーズのルール

5. 合法な配置アクションがない場合、`EndTurnAction` があれば終了する。

6. 配置候補ごとに、配置後に自分が受け取るマイナスカード枚数を計算する。

7. 受け取るマイナスカード枚数が最小の配置を優先する。

   理由: 決算時にマイナスカード1枚につき失点チャートが1進むため、まず直接的な失点リスクを避ける。

8. その中で、プラスポイントによる失点チャート減少が最大の配置を優先する。

   理由: 失点チャートを下げる手段はプラスポイントだけなので、取れる減少は優先する。

9. その中で、真ん中の数字から遠いカードを優先して出す。

   距離は `abs(rank_index - 3)` とする。`rank_index` が 0 または 6 のカードを最も優先し、3 のカードを最も後回しにする。

   理由: 真ん中から遠い数字は配置できる行が端に寄り、後から置きにくくなる可能性が高い。早めに処理する。

10. その中で、重ね置きになる配置を優先する。

    理由: 重ね置きでは3x3枠内の占有マス数が増えないため、他プレイヤーがプラスポイントで失点チャートを下げるチャンスを減らせる。

11. その中で、自分の手札に同じ色のカードが多く残る列へ置ける配置を優先する。

    理由: 同じ色は同じ列にしか置けないため、同色カードを続けて処理しやすい列を維持する。

12. その中で、現在の盤面上に多い色のカードを出す。

    理由: 盤面上に多い色は、その色の列が次の自分の手番まで残っていない可能性がある。後で出せなくなる前に処理する。

13. その中で、手札 index が小さいカードを優先する。

14. 1枚目を置いた後、手札にまだ配置可能カードがあり、2枚目候補の最小マイナスカード枚数が0なら2枚目を置く。

15. 1枚目を置いた後、2枚目候補でプラスポイントを得られるなら2枚目を置く。

16. 1枚目を置いた後、2枚目候補の最小マイナスカード枚数が1以上で、プラスポイントもないなら1枚で終了する。

17. 2枚目を置く場合も、候補選択はルール6〜13と同じ評価で決める。

18. すべての評価が同点なら、固定 tie-break で1つ選ぶ。

## 実装メモ

- 配置候補の評価は、候補アクションを仮適用した `GameState` から計算する。
- マイナスカード枚数は、行動前後の現在プレイヤーの `negative_cards` 差分で評価する。
- プラスポイントは、行動前後の現在プレイヤーの `loss_score` 差分で評価する。
- 重ね置き判定は、配置前の `board` に `action.position` が存在するかで判定する。
- 盤面上の色の多さは、重ね置きされたカードも含めたカード枚数で数える。
- 真ん中からの距離は `abs(card.rank_index - 3)` で計算する。
- 実装先は `src/yellowstone/bots.py` とし、最初は `choose_heuristic_action(state) -> Action | None` の形にする。
