# Yellowstone Park Rules Design

## Game Overview

Yellowstone Park は、4〜5人で遊ぶカード配置ゲーム。
7x7 のボードに、色と数字を持つカードを配置していく。
プレイヤーはマイナスカードをできるだけ引き取らないようにしながら、失点チャートの進行を抑える。
誰かが失点チャート35以上になった決算でゲーム終了し、もっとも失点チャートが進んでいないプレイヤーが勝つ。

## Components

- ボード: 7x7 マス
- 失点チャート: 0 から 35 以上まで進む失点トラック
- カード: 4色 x 数字 1〜7 x 各2枚、合計56枚
- 山札
- 各プレイヤーの手札
- 各プレイヤーのマイナスカード置き場

## Player Count

このオンライン版では、まず 4〜5人プレイのみを対象にする。

2〜3人プレイ用の「2枚配置後は必ず手札補充する」ルールは、初期実装では対象外とする。

## Setup

1. 56枚のカードをシャッフルして山札を作る。
2. 各プレイヤーに手札を6枚配る。
3. 各プレイヤーの失点チャートを開始失点に置く。標準値は5。
4. 山札から1枚めくる。
5. めくったカードを、そのカードの数字と同じ横列の中央に置く。

オンライン版では、開始失点をゲーム開始前に変更できるようにする。
開始後は開始失点を変更できない。

## Board Coordinates

ボードは 7x7 の座標で表現する。

- `x`: 横方向の位置。0〜6。
- `y`: 縦方向の位置。0〜6。
- 座標 `(0, 0)` は右上のマスとする。
- 横列の数字は `y` に対応する。
- 各横列の中央は `x=3` とする。

## Frame Coordinates

3x3 の枠は、右上座標で管理する。

右上座標として指定できるのは、3x3 の枠がボード内に収まる座標だけ。
7x7 ボードでは、右上座標の有効範囲は以下とする。

- `x`: 0〜4
- `y`: 0〜4

例:

- `(0, 0)` は有効。右上から始まる 3x3 枠。
- `(4, 4)` は有効。左下寄りの 3x3 枠。
- `(5, 3)` は無効。横方向に 3マス分の枠を作れない。
- `(4, 5)` は無効。縦方向に 3マス分の枠を作れない。

3x3 枠の候補数は、通常ルールでは 25 個。

## Board Placement Rules

### Row Rule

数字カードは、ボード横に表示された数字と同じ横列にしか置けない。

例: 数字 3 のカードは 3 の横列にだけ置ける。

### Column Color Rule

縦列は、その列に最初に置かれたカードの色で固定される。
以降、その縦列には同じ色のカードしか置けない。

すでにボード上のどこかの縦列に同じ色のカードが存在する場合、その色のカードはその既存の色列にしか置けない。
同じ色のカードを、別の未確定列や別の色で固定済みの列に置くことはできない。

ただし、その縦列からカードがすべて無くなった場合、その縦列は未確定に戻る。
その後は、次に置かれたカードの色で新しく固定される。

## Turn Flow

手番は、プレイフェーズと手札補充フェーズに分けて処理する。

### Play Phase

手番プレイヤーは、手札からカードを1枚または2枚置く。

#### Place 1 Card

- カードを1枚置く。
- 1枚だけで手番を終える場合は、`1枚で終了` を選ぶ。

#### Place 2 Cards

- カードを2枚置く。
- 1枚目のカードプレイ処理が完了した後、手札にカードが残っていれば、追加の選択ボタンを押さずに2枚目をプレイできる。
- 2枚目をプレイしない場合だけ、`1枚で終了` を選ぶ。

### Undo To Turn Start

オンライン版では、現在の手番を開始時点に戻す操作を用意する。
これは実物ゲームのルールではなく、ローカルUIでの誤操作を戻すための補助機能とする。

`ターン開始時に戻る` を押せるタイミング:

- 1枚目のカードプレイ処理が完了し、まだ `1枚で終了` を押していないとき。
- 2枚目のカードプレイ処理が完了し、まだ補充操作を選んでいないとき。
- 2枚プレイ後の手札補充フェーズで、`山札から補充`、`マイナスカードから補充`、`補充しない` のいずれも選んでいないとき。

`ターン開始時に戻る` を押せないタイミング:

- その手番でまだカードを1枚もプレイしていないとき。
- 3x3枠選択中など、カードプレイ処理がまだ確定していないとき。この場合は `キャンセル` を使う。
- `1枚で終了` を押して次プレイヤーへ進んだ後。
- 補充操作または補充しない操作を選んで次プレイヤーへ進んだ後。
- 決算処理中またはゲーム終了後。

### Refill Phase

手札補充フェーズに移行できるのは、2枚プレイした場合だけ。
1枚だけプレイした場合は、手札補充フェーズに移行せず、次のプレイヤーへ進む。

手札が0枚でない場合:

- 以下のどちらかを選ぶ。
  - 山札から手札が6枚になるよう補充する。
  - 補充しない。

手札が0枚の場合:

- 基本は山札から6枚補充する。
- ただし、引き取ったマイナスカードが6枚以上ある場合、そのマイナスカードをシャッフルし、そこから6枚引いて手札にできる。
- マイナスカードから補充できる場合でも、山札から補充することを選べる。
- マイナスカードから6枚補充した場合、その6枚はマイナスカード置き場から取り除く。

## Overflow And Negative Cards

ボード上では、配置により 3x3 の枠を評価する。
カードをプレイしたプレイヤーは、評価する 3x3 の枠を1つ選ぶ。
選ぶ 3x3 の枠は、今置いたカードを含んでいなければならない。

選べる枠は1つだけ。
選んだ 3x3 の枠からはみ出した分のカードは、マイナスカードとして現在の手番プレイヤーが引き取る。
このとき、はみ出したカードだけをボードから取り除き、選んだ枠内のカードはボード上に残す。

引き取ったマイナスカードは裏向きで保持する。

決算時、マイナスカード1枚につき失点チャートを1進める。

各プレイヤーのターン開始時点では、ボード上のすべてのカードが、少なくとも1つの有効な3x3枠の内側に収まっていなければならない。
カードプレイ処理は、選んだ3x3枠からはみ出したカードを直ちに回収するため、処理完了後の盤面は常にこの条件を満たす。

## Positive Points

3x3 の枠に対して、配置したカードが以下の条件を満たす場合、失点チャートを戻す。

- 8マス目に置いた: 失点チャートを1下げる
- 9マス目に置いた: 失点チャートを3下げる

プラスポイントは、今置いたカードによって選択中の 3x3 枠内の占有マス数が初めて8マス目または9マス目になった場合にだけ発生する。
既存カードがすでにあるマスへ配置することはできるが、その配置でプラスポイントを得ることはできない。
失点チャートは0未満にはならない。
失点チャートを下げる方法は、このプラスポイント処理のみ。

## Scoring Settlement

山札が切れるたび、手札補充フェーズに決算を挟む。

決算では、全プレイヤーが自分のマイナスカード枚数分だけ失点チャートを進める。
山札が不足していた補充分は補充しない。

決算後、ゲーム終了条件を満たしていなければ、ボードと各プレイヤーの手札はそのまま維持してゲームを続行する。

1. 全プレイヤーのマイナスカードをすべて回収する。
2. 回収したカードをシャッフルし、新しい山札を作る。
3. 最後に手札補充したプレイヤーの次のプレイヤーからプレイを再開する。

ボード上のカードは山札に戻さない。
この設計では、ラウンドという独立した区切りは設けない。

## Game End

決算後、誰かが失点チャート35以上になっていたらゲーム終了。

もっとも失点チャートが進んでいないプレイヤーが勝利する。

もっとも失点チャートが進んでいないプレイヤーが複数いる場合、そのプレイヤー全員が勝利を分かち合う。

## Advanced Rules

上級ルールでは、通常の3x3枠に加えて、以下の追加枠を使う。

- 1 の列 3マス
- 7 の列 3マス
- 上記がつながった 3x2 の枠

追加枠に対するプラスポイント:

- 5マス目に置いた: 失点チャートを1下げる
- 6マス目に置いた: 失点チャートを3下げる

## 間違えやすいルール

- このオンライン版では、まず 4〜5人プレイのみを対象にする。
- 「スコアチャート」ではなく「失点チャート」と呼ぶ。
- 開始失点の標準値は5だが、オンライン版ではゲーム開始前に変更できる。開始後は変更できない。
- 3x3 枠は右上座標で管理する。7x7 ボードで有効な右上座標は `x=0..4`, `y=0..4`。
- 初期配置カードは、該当する横列の中央 `x=3` に置く。
- 各プレイヤーのターン開始時、ボード上の全カードは1つの有効な3x3枠内に収まっていなければならない。
- UI確認用fixtureやテスト用の初期状態も、特別に不正状態を検証する場合を除き、3x3枠内に収まる盤面として作る。
- はみ出し判定に使う 3x3 枠は、カードをプレイしたプレイヤーが1つだけ選ぶ。
- 選ぶ 3x3 枠は、今置いたカードを含んでいなければならない。
- はみ出したカードだけをボードから取り除く。選んだ 3x3 枠内のカードは残す。
- プラスポイントは、今置いたカードによって選択中の 3x3 枠内の占有マス数が初めて8マス目または9マス目になった場合にだけ発生する。
- 既存カードがあるマスにも配置できる。ただし、その配置でプラスポイントを得ることはできない。
- すでにボード上にある色のカードは、その色が置かれている既存の縦列にしか置けない。未確定列に同じ色の2列目を作ることはできない。
- 手番はプレイフェーズと手札補充フェーズに分ける。
- 1枚目のプレイ後、`2枚目をプレイ` ボタンは使わない。`1枚で終了` を押さない限り、手札からそのまま2枚目を選べる。
- `ターン開始時に戻る` は、現在手番の確定済み操作をターン開始時点へ戻す補助機能。次プレイヤーへ進んだ後や補充操作後には押せない。
- 手札補充フェーズに移行できるのは、2枚プレイした場合だけ。
- 1枚だけプレイした場合は、手札が0枚になっても補充できない。
- 手札が0枚になった場合、基本は山札から6枚補充する。
- 手札が0枚で、マイナスカードが6枚以上ある場合は、マイナスカードをシャッフルして6枚を手札にできる選択肢が追加される。
- マイナスカードから補充できる場合でも、山札から補充することを選べる。
- マイナスカードから補充した6枚は、マイナスカード置き場から取り除く。
- 山札切れはラウンド終了ではなく、手札補充フェーズ中に挟まる決算として扱う。
- 決算後も、ボードと手札はそのまま維持する。
- 決算後に新しい山札を作るとき、ボード上のカードは山札に戻さない。
- 決算後は、最後に手札補充したプレイヤーの次のプレイヤーから再開する。
- 同点の場合は、該当プレイヤー全員が勝利を分かち合う。

## Strategy Notes

- 手札0枚時にマイナスカード6枚から補充すると、その6枚はマイナスカード置き場から取り除かれる。結果として、次の決算時に数えるマイナスカードが6枚減るため、実質的に6点分の失点を避けられる。
- ただし、マイナスカードから補充できる場合でも、山札から補充する方がよいことがある。
- 例: 現在の盤面が数字1〜3中心なのに、マイナスカードに数字5〜7が多い場合、マイナスカードから補充すると置きにくい手札になりやすい。
- その場合、一時的に6点分の失点を避けられても、後で大量失点につながる恐れがある。

## Implementation Notes

- カードは `color` と `rank` を持つデータとして扱う。
- 強化学習や配列処理で扱いやすいよう、内部表現では `rank_index` を 0〜6 として持ってよい。
- ルール説明や表示上の数字は `rank = rank_index + 1` として扱う。
- ボードは 7x7 の座標で表現する。
- 横列の数字制限は、内部表現では `rank_index == y` として判定する。
- 既存カードがあるマスへの配置は重ね置きとして扱い、既存カードは取り除かない。
- 既存カードがあるマスへの配置では、そのマスはすでに占有済みなので、3x3枠内の占有マス数は増えない。
- 縦列の色制限は、その列に存在するカード色と、ボード上に存在する同色カードの列から判定する。
- すでにボード上に存在する色は、その色が存在する列だけを配置候補にする。
- 縦列が空になった場合、その列の色制限は解除される。
- 3x3 枠は右上座標で表現し、有効範囲は `x=0..4`, `y=0..4` とする。
- カードプレイ時に選ぶ 3x3 枠は、今置いたカードを含む必要がある。
- ターン開始処理では、ボード上の全カードが少なくとも1つの有効な3x3枠に収まることを検証する。
- 初期配置カードは、該当する横列の中央 `x=3` に置く。
- 手番処理は、プレイフェーズと手札補充フェーズを明確に分ける。
- 決算は、独立したラウンド終了ではなく手札補充フェーズ中のイベントとして扱う。
- ゲームロジックは React から分離し、純粋関数で実装する。
