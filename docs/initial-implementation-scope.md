# Initial Implementation Scope

この文書は `docs/game-rules-design.md` を、最初に Python 実装へ落とすための対象範囲として整理する。

目的は、人間向け UI ではなく、テキストベースの開発者用確認、自己対戦、探索、強化学習へ進めるためのゲーム環境を作ること。

## 対象にするルール

### プレイヤー数

- 4〜5人プレイのみ対応する。
- 2〜3人用の補充ルールは実装しない。
- 不正なプレイヤー数は初期化時にエラーにする。

### カードと山札

- カードは `color` と `rank_index` を持つ。
- `rank_index` は 0〜6 の内部表現とし、表示やルール説明上の数字は `rank = rank_index + 1` とする。
- 色は4色、数字は1〜7、各組み合わせ2枚、合計56枚。
- 初期化時に山札をシャッフルする。
- 乱数は外から seed または RNG を渡せる形にする。

### 初期化

- 各プレイヤーに6枚ずつ配る。
- 各プレイヤーの失点チャートは標準5から開始する。
- 開始失点は初期化引数で変更できる。
- 山札から1枚めくり、そのカードの数字と同じ横列の中央 `x=3` に置く。
- 初期化後の手番プレイヤーは `0` とする。

### 座標と盤面

- ボードは7x7。
- 座標は `(x, y)` で表す。
- `x` は 0〜6、`y` は 0〜6。
- `(0, 0)` は右上。
- 内部表現では、カードの `rank_index` と同じ `y` の横列にだけ置ける。
- 表示上は `rank 1 -> y=0`, `rank 7 -> y=6` に対応する。
- 各行の中央は `x=3`。

### 3x3枠

- 通常ルールの3x3枠のみ対象にする。
- 枠は右上座標 `(x, y)` で表す。
- 有効な右上座標は `x=0..4`, `y=0..4`。
- カード配置後、配置したカードを含む枠を1つ選ぶ。
- 選んだ枠の外にあるカードは、現在プレイヤーのマイナスカードになる。
- 選んだ枠の内側にあるカードは盤面に残る。
- 各ターン開始時点で、盤面上の全カードは少なくとも1つの有効な3x3枠に収まっている必要がある。

### 配置ルール

- 数字カードは、数字と同じ `y` の横列にのみ置ける。
- 既存カードがあるマスにも配置できる。
- 既存カードがあるマスへ配置した場合は重ね置きとし、既存カードは上書きしない。
- 既存カードがあるマスへの配置では、そのマスはすでに占有済みなので、3x3枠内の占有マス数は増えない。
- 縦列は、その列に存在するカード色で固定される。
- 空列は色未確定として扱う。
- すでに盤面上に存在する色のカードは、その色が存在する既存列にしか置けない。
- 同じ色の2列目を未確定列に作ることはできない。
- 列が空になった場合、その列の色固定は解除される。

### 手番

- 手番プレイヤーは手札から1枚または2枚置く。
- 1枚だけ置いた場合、補充せずに次プレイヤーへ進む。
- 2枚置いた場合のみ補充フェーズへ進む。
- 1枚目の配置処理完了後、手札が残っていれば2枚目を置ける。
- 2枚目を置かない場合は1枚で終了する。

### 補充

- 補充フェーズは2枚置いた場合だけ発生する。
- 手札が1枚以上なら、山札から6枚になるまで補充するか、補充しないかを選ぶ。
- 手札が0枚なら、基本は山札から6枚補充する。
- 手札が0枚かつマイナスカードが6枚以上ある場合、マイナスカードから6枚補充する選択肢もある。
- マイナスカードから補充した6枚は、マイナスカード置き場から取り除く。
- マイナスカードから補充できる場合でも、山札から補充できる。

### プラスポイント

- 選択した3x3枠内の占有マス数が、今置いたカードによって初めて8になった場合、現在プレイヤーの失点チャートを1下げる。
- 初めて9になった場合、現在プレイヤーの失点チャートを3下げる。
- 失点チャートは0未満にしない。
- 既存カードがあるマスへの配置ではプラスポイントを得ない。

### 決算

- 山札が補充中に尽きた場合、補充フェーズ中に決算する。
- 山札が足りなかった分は補わない。
- 全プレイヤーは、自分のマイナスカード枚数だけ失点チャートを進める。
- 決算後、誰かが35以上ならゲーム終了。
- ゲームが続く場合、全プレイヤーのマイナスカードを集めてシャッフルし、新しい山札にする。
- ボード上のカードと各プレイヤーの手札は維持する。
- 最後に補充したプレイヤーの次のプレイヤーから再開する。

### 勝敗

- 決算後、誰かの失点チャートが35以上なら終了する。
- 失点チャートが最も低いプレイヤーが勝つ。
- 同点の場合は該当プレイヤー全員を勝者にする。

## 初期実装では対象外にするもの

- 2〜3人プレイ用ルール。
- 上級ルールの追加枠。
- Web UI。
- Undo to turn start。
- 画像や視覚的な UI。
- ニューラルネットワークによる強化学習。
- 高度な探索 bot。

## 実装順序

1. `types.py` に Card、Color、Position、Frame、PlayerState、GameState、Action 系の基本型を定義する。
2. `game.py` にデッキ生成、初期化、枠判定、配置可能マス判定を実装する。
3. 1枚配置後に選べる3x3枠と、枠外カード回収を実装する。
4. 1枚終了、2枚配置、補充選択を含むターン進行を実装する。
5. 山札切れ決算と勝敗判定を実装する。
6. `render.py` にテキスト盤面表示を実装する。
7. `cli.py` から手動で状態確認と簡易プレイができるようにする。
8. `bots.py` に random bot を追加する。

## テスト計画

- デッキが56枚で、各カードが2枚ずつ存在する。
- 4〜5人以外では初期化できない。
- 初期手札が各6枚配られ、初期カードが該当行の中央に置かれる。
- 内部 `rank_index` と `y` が一致する横列にだけ配置できる。
- 数字と異なる横列には置けない。
- 列色ルールに反する配置はできない。
- 同色カードの2列目を作れない。
- 空になった列は色未確定に戻る。
- 有効な3x3枠は25個だけ。
- 配置カードを含まない枠は選べない。
- 枠外カードは現在プレイヤーのマイナスカードになる。
- 8マス目、9マス目のプラスポイントが正しく失点チャートを下げる。
- 既存カードがあるマスへの配置ではプラスポイントが発生しない。
- 既存カードがあるマスへの配置は重ね置きになり、占有マス数は増えない。
- 1枚終了では補充が発生しない。
- 2枚配置後だけ補充できる。
- 山札切れ時に決算が発生し、マイナスカード枚数ぶん失点する。
- 決算後に35以上のプレイヤーがいればゲーム終了する。
- 同点勝利者を複数返せる。

## 確定した設計判断

- 既存カードがあるマスへ配置したときは、同じマスに重ねて複数枚保持する。
- 既存カードがあるマスへ配置した場合、枠内占有マス数は増えない。
- 内部表現では `rank_index 0 -> y=0`, `rank_index 6 -> y=6` とする。
- 表示やルール説明上は `rank = rank_index + 1` として、`rank 1 -> y=0`, `rank 7 -> y=6` に対応させる。
